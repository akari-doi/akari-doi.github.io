WordPressのページ表示の仕組み（テンプレート階層）

目的: WordPressがどのテンプレートファイルを選んでページを表示するかを理解する
使用テーマ: Twenty Seventeen
環境: Local（ローカル環境）

1. WordPressページ表示の基本

1.1 WordPressはどうやってページを作っているの？

通常のHTMLサイトとWordPressサイトの違い：

通常のHTMLサイト

about.html ← 直接このファイルが表示される
contact.html ← 直接このファイルが表示される
index.html ← 直接このファイルが表示される

WordPressサイト

ユーザーがURLにアクセス
↓
WordPressが「どのページを表示するか」を判断
↓
適切なテンプレートファイルを選択
↓
データベースから内容を取得
↓
テンプレートファイル + データ = 完成したページ

1.2 テンプレートファイルとは

テンプレートファイル = ページの「型」を決めるPHPファイル

例えば：

  • index.php = トップページの型
  • single.php = ブログ記事ページの型
  • page.php = 固定ページの型

WordPressは「今表示しようとしているのはどんなページか？」を判断して、適切なテンプレートファイルを自動で選びます。

2. テンプレート階層とは

2.1 優先順位のルール

WordPressは決まった順番でテンプレートファイルを探します。見つかったら、それを使ってページを表示します。

より具体的なファイル
↓（なければ）
やや具体的なファイル
↓（なければ）
一般的なファイル
↓（なければ）
index.php（必ず存在するフォールバック）

2.2 実例：ブログ記事ページの場合

投稿タイプ「post」、スラッグ「hello-world」の記事を表示する場合：

1. single-post-hello-world.php（投稿タイプ+スラッグ指定）← 最優先
↓（なければ）
2. single-post.php（投稿タイプ指定）
↓（なければ）
3. single.php（一般的なブログ記事）
↓（なければ）
4. singular.php（投稿・固定ページ共通）
↓（なければ）
5. index.php（フォールバック）← 必ず存在

3. 主要なテンプレートファイル

3.1 必須ファイル

ファイル名 用途 重要度

5. 準備：環境とツールのセットアップ

5.1 Local環境の起動

まずはローカル環境を立ち上げましょう

  1. Localアプリケーションを起動
  2. 「wp-site」を選択
  3. 「Start site」ボタンをクリック
  4. 「WP Admin」ボタンをクリック - WordPress管理画面にログイン

5.2 Show Current Templateプラグインのインストール

テンプレート階層を視覚的に確認できる便利なプラグインを使用します

  1. WordPress管理画面の左メニューから プラグイン > 新規追加 をクリック
  2. 検索ボックスに「Show Current Template」と入力
  3. 該当するプラグインの 「今すぐインストール」 をクリック
  4. インストール完了後、「有効化」 をクリック

プラグインの機能：

  • 現在表示中のページでどのテンプレートファイルが使われているかが分かる
  • ファイルパスも表示される
  • リアルタイムでテンプレート階層を確認できる

5.3 Twenty Seventeenテーマのインストール

手順：

  1. WordPress管理画面 > 外観 > テーマ
  2. 「新しいテーマを追加」をクリック
  3. 「Twenty Seventeen」を検索
  4. インストール・有効化

6. 実際にファイルを確認してみよう

6.1 ファイル確認

ローカル環境でのファイル確認方法：

wp-site/ └── app/ └── public/ └── wp-content/ └── themes/ └── twentyseventeen/ ├── 404.php ├── archive.php ├── front-page.php ├── page.php ├── single.php ├── search.php ├── index.php ├── header.php ├── footer.php ├── functions.php └── style.css

※主なファイルだけ書き出しています。実際にはもっと多くのファイルが存在します。

確認方法：

Localを使って開く: Localの「Site folder」をクリックしてwp-siteフォルダを開き、上記のようにフォルダをたどっていき、twentyseventeenフォルダを開く

Twenty Seventeenで確認すべきファイル：

手順2： Show Current Templateプラグインでページごとのテンプレートを確認

6.2 Show Current Templateでの実践確認

各ページで実際に確認してみましょう：

  1. フロントページ → front-page.php が表示される（Twenty Seventeenはカスタマイズにより例外あり）
  2. ブログ記事ページ → single.php が表示される
  3. 固定ページ → page.php が表示される
  4. 存在しないURL → 404.php が表示される

7. まとめ

テンプレート階層の重要ポイント

WordPressは自動でファイルを選ぶ

  • 決まった順番でファイルを探す
  • 見つかったファイルを使ってページを表示

具体的 → 一般的の順番

  • より具体的なファイルが優先される
  • 最終的には index.php がフォールバック

ファイル名には意味がある

  • page-about.php = 「about」スラッグの固定ページ専用
  • category-news.php = 「news」カテゴリー専用
  • single.php = すべてのブログ記事

実際のファイル確認が重要

  • テーマによって存在するファイルは異なる
  • 自分で作成することもできる

Show Current Templateで確認できる

  • 今表示されているページがどのファイルを使っているかがわかる
